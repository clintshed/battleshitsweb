Battle Shits – README (updated)
What is Battle Shits?

Battle Shits is a tongue‑in‑cheek, browser‑based remake of the classic “Battleship” game – only every ship is a shit and every hit is a glorious plop of brown. The game runs entirely in the browser (HTML + CSS + JavaScript) and can be hosted on GitHub Pages with a single click.
The (very real) origin story

Back in 2020 I completely forgot to vote in the election. My friend, still angry and starving, ordered a mountain of Taco Bell, hopped on his skateboard, and rolled past my house – leaving a full‑scale poop‑storm on my front yard.  

A few weeks later I stumbled across an old Pascal program called battleshits that already featured “shits” instead of ships. The code even printed messages like “You hit a shit!” when a hit occurred . The coincidence was uncanny: the game’s crude humor matched my own yard‑shitting mis‑adventure perfectly.

That serendipity made me decide to re‑engineer the whole thing in HTML/JavaScript so anyone could play it in a modern browser. I used Vibe Coding for the quick scaffolding and leaned on an AI assistant (yes, even smarter than Kira) to clean up the logic, add an intro video, fart sound effects, and drag‑and‑drop poops. The result is the polished web version you see today.
Features

    10 × 10 grids for you and the bot (5‑, 4‑, 3‑, 3‑, 2‑length shits).  
    Random placement that mirrors the original Pascal Addboat logic.  
    Turn‑based play – you fire, the bot fires, until one side runs out of shits.  
    Intro video that plays on first load (or can be skipped).  
    Fart / poop sound effects (assets/fart.mp3).  
    Drag‑and‑drop poops for pure visual comedy.
